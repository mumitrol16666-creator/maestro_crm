# 🔧 Исправление создания менеджера/пользователя

## Дата: 12 октября 2025, 21:40

## Проблемы

### 1. Ошибка сервера при создании менеджера
```
POST /api/users/sales-managers [500]
Student validation failed: gender: Пол обязателен
```

### 2. HTML код в уведомлении
```html
<div style="display: flex; align-items: flex-start...">
  <div style="flex-shrink: 0...">
    <svg width="24"...>
```

Вместо красивого уведомления показывались HTML теги.

## Причины

### 1. Отсутствие поля gender
При создании менеджера/преподавателя/админа не передавалось обязательное поле `gender`, которое требуется в модели Student.

**Код:**
```javascript
let body = { name, phone, password };
// gender отсутствует! ❌
```

### 2. Дублирующаяся функция showNotification
В `admin.js` была локальная версия `showNotification` (строка 136), которая использовала `textContent` вместо `innerHTML`:

```javascript
function showNotification(message) {
    notification.textContent = message; // ❌ Показывает HTML как текст
}
```

Это переопределяло глобальную функцию из `script.js`.

## Решение

### 1. Добавлено поле gender

**Файл:** `frontend/js/admin.js`

```javascript
let body = { 
    name, 
    phone, 
    password,
    gender: 'male' // ✅ Добавлено обязательное поле
};
```

**Почему 'male'?**
- Это технический пользователь (менеджер/админ/преподаватель)
- Пол не важен для не-учеников
- Поле обязательное в модели, поэтому ставим значение по умолчанию

### 2. Удалена дублирующаяся функция

**До:**
```javascript
function showNotification(message, type = 'success') {
    notification.textContent = message; // Локальная версия
}
```

**После:**
```javascript
// Примечание: showNotification определена в script.js
// Эта локальная версия удалена, чтобы использовать глобальную
```

Теперь используется глобальная функция из `script.js`, которая поддерживает HTML.

### 3. Создание пользователя показывает модальное окно

**До:**
```javascript
showNotification(`Пользователь создан!\n\nТелефон: ${phone}\nПароль: ${password}`);
```

**После:**
```javascript
const userTypeText = {
    'sales_manager': 'Менеджер по продажам',
    'teacher': 'Преподаватель',
    'admin': 'Администратор'
}[role];

showPasswordModal(name, phone, password, copySuccess, userTypeText);
```

### 4. Обновлена функция showPasswordModal

Добавлен параметр `userType`:

```javascript
function showPasswordModal(userName, userPhone, password, copySuccess, userType = '') {
    const title = userType ? `${userType.toUpperCase()} СОЗДАН` : 'НОВЫЙ ПАРОЛЬ СОЗДАН';
    // ...
}
```

**Заголовки:**
- Создан менеджер → "МЕНЕДЖЕР ПО ПРОДАЖАМ СОЗДАН"
- Создан преподаватель → "ПРЕПОДАВАТЕЛЬ СОЗДАН"
- Создан админ → "АДМИНИСТРАТОР СОЗДАН"
- Сброс пароля → "НОВЫЙ ПАРОЛЬ СОЗДАН"

### Почему было две функции showNotification?

1. **Глобальная** (script.js):
   - Поддерживает HTML (`innerHTML`)
   - Поддерживает темы
   - Иконки отображаются

2. **Локальная** (admin.js - удалена):
   - Использовала `textContent`
   - Переопределяла глобальную
   - HTML показывался как текст
