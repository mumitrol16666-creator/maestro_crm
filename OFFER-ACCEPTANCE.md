# 📋 Обязательное принятие публичной оферты

## Дата: 12 октября 2025, 22:30

## Что реализовано

### ✅ Обязательное принятие оферты при первом входе

При первом входе в профиль ученик **обязательно** должен принять публичную оферту, чтобы продолжить работу.

## Как работает

### Процесс первого входа:

```
1. Ученик входит в профиль
   ↓
2. Система проверяет: offerAccepted === false?
   ↓
3. ДА → Показывается модальное окно с офертой
   - Кнопка × скрыта (нельзя закрыть!)
   - Клик на overlay не работает
   - Форма согласия видна
   ↓
4. Ученик читает текст оферты
   ↓
5. Ставит галочку "Я ознакомлен и согласен"
   ↓
6. Активируется кнопка "ПРИНЯТЬ И ПРОДОЛЖИТЬ"
   ↓
7. Нажатие кнопки → POST /api/students/:id/accept-offer
   ↓
8. Сервер сохраняет:
   - offerAccepted: true
   - offerAcceptedAt: текущая дата
   ↓
9. Окно закрывается
   ↓
10. Ученик может работать с профилем
```

### Повторный вход:

```
1. Ученик входит в профиль
   ↓
2. Система проверяет: offerAccepted === true?
   ↓
3. ДА → Оферта не показывается
   ↓
4. Профиль загружается нормально
```

### Кнопка "ОФЕРТА" в меню:

```
1. Ученик нажимает кнопку "ОФЕРТА"
   ↓
2. Открывается окно для ознакомления
   - Кнопка × видна (можно закрыть)
   - Клик на overlay работает
   - Форма согласия скрыта (уже принято)
   ↓
3. Ученик читает
   ↓
4. Закрывает окно (×, overlay, ESC)
```

## Дополнительные возможности

### 1. Middleware для проверки оферты

**Файл:** `backend/src/middleware/auth.js`

```javascript
const requireOfferAcceptance = async (req, res, next) => {
    if (req.user.role === 'student' && !req.user.offerAccepted) {
        return res.status(403).json({
            error: 'Необходимо принять публичную оферту'
        });
    }
    next();
};
```

Использование:
```javascript
router.post('/some-route', authenticate, requireOfferAcceptance, ...)
```
