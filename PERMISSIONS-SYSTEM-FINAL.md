# ✅ Интерактивная система управления правами - ГОТОВО!

## 🎯 Что создано

Полноценная система управления правами доступа с реальным управлением функциями и видимостью разделов.

## 📊 Основные компоненты

### 1. Интерактивная таблица управления правами

**Две таблицы:**

#### A. ФУНКЦИОНАЛЬНЫЕ ПРАВА
Управление тем, что может делать каждая роль:
- Управление заявками
- Управление учениками
- Просмотр учеников
- Управление группами
- Просмотр групп
- Управление абонементами
- Подтверждение платежей
- Просмотр платежей
- Создание преподавателей
- Редактирование преподавателей
- Удаление преподавателей
- Управление администраторами
- Отметка посещаемости
- Управление практиками
- Управление направлениями
- Управление залами
- Системные настройки

#### B. ВИДИМОСТЬ РАЗДЕЛОВ В АДМИНКЕ
Управление тем, какие разделы видит каждая роль:
- Дашборд
- Заявки
- Ученики
- Группы
- Абонементы
- Практики
- Расписание
- Направления
- Пользователи
- Управление ролями

## 🎨 Интерфейс

### Статусы прав:
- **ON** (зеленый) - Право включено
- **OFF** (серый) - Право выключено
- **LOCKED** (желтый) - Заблокировано для Super Admin

### Интерактивность:
1. **Клик на ON** → Выключается (становится OFF красным при hover)
2. **Клик на OFF** → Включается (становится ON зеленым при hover)
3. **LOCKED** - Нельзя кликнуть (права Super Admin всегда полные)

### Автосохранение:
- Каждый клик сохраняется на сервер автоматически
- При ошибке - откат изменений
- Уведомление об успехе/ошибке

## 🔧 Backend

### Модель: `RolePermissions.js`

```javascript
{
    role: String (unique),
    permissions: {
        manageBookings: Boolean,
        manageStudents: Boolean,
        viewStudents: Boolean,
        // ... и т.д.
    },
    visibility: {
        dashboard: Boolean,
        bookings: Boolean,
        students: Boolean,
        // ... и т.д.
    }
}
```

### API Endpoints:

#### 1. GET /api/permissions
Получить права всех ролей
- **Доступ:** Admin, Super Admin
- **Возвращает:** Массив прав для всех ролей

#### 2. PATCH /api/permissions/:role
Обновить права роли
- **Доступ:** Только Super Admin
- **Body:** `{ permissions: {...}, visibility: {...} }`
- **Защита:** Нельзя изменить права Super Admin

#### 3. POST /api/permissions/:role/reset
Сбросить права к дефолтным
- **Доступ:** Только Super Admin
- **Действие:** Возвращает дефолтные права роли

### Дефолтные права:

#### Super Admin (LOCKED):
- ✅ Все функции: ON
- ✅ Все разделы: ON
- 🔒 Нельзя изменить

#### Admin:
- ✅ Почти все функции
- ❌ НЕТ: удаление преподов, управление админами, направления, системные настройки
- ✅ Видит: дашборд, заявки, ученики, группы, абонементы, практики, расписание, пользователи, роли
- ❌ НЕ видит: направления

#### Sales Manager:
- ✅ Управление заявками
- ✅ Просмотр учеников и групп
- ❌ НЕТ: управление финансами, абонементами
- ✅ Видит: дашборд, заявки, ученики, группы
- ❌ НЕ видит: абонементы, практики, расписание, направления, пользователи, роли

#### Teacher:
- ✅ Просмотр учеников и групп
- ✅ Отметка посещаемости
- ❌ НЕТ: управление чем-либо
- ✅ Видит: дашборд, ученики, группы, практики, расписание
- ❌ НЕ видит: заявки, абонементы, направления, пользователи, роли

#### Student:
- ❌ Все функции: OFF
- ❌ Все разделы админки: OFF
- (Student использует профиль, а не админку)

## 💻 Frontend

### HTML: `/frontend/public/admin.html` (v=208)

Секция "Управление ролями":
```html
<div class="permissions-matrix-interactive">
    <h3>УПРАВЛЕНИЕ ПРАВАМИ РОЛЕЙ</h3>
    
    <!-- Таблица функциональных прав -->
    <table id="permissionsTable"></table>
    
    <!-- Таблица видимости разделов -->
    <table id="visibilityTable"></table>
    
    <!-- Кнопка сброса -->
    <button onclick="resetPermissionsToDefault()">
        Сбросить к дефолтным
    </button>
</div>
```

### JavaScript: `/frontend/js/admin.js`

**Функции:**
- `loadRolesData()` - Загрузка прав с сервера
- `renderPermissionsTable()` - Отрисовка таблицы функций
- `renderVisibilityTable()` - Отрисовка таблицы видимости
- `togglePermission(role, type, key)` - Переключение права (с автосохранением)
- `resetPermissionsToDefault()` - Сброс всех прав
- `getRoleNameShort()` - Короткие названия ролей

**Словари:**
- `permissionLabels` - Названия функций на русском
- `visibilityLabels` - Названия разделов на русском
- `rolePermissions` - Объект с правами всех ролей

### CSS: `/frontend/css/admin-styles.css`

**Классы:**
- `.permissions-matrix-interactive` - Контейнер
- `.interactive-table` - Таблица
- `.perm-status` - Бейдж статуса (ON/OFF/LOCKED)
- `.access-cell.clickable` - Кликабельная ячейка
- `.access-cell.locked` - Заблокированная ячейка
- `.matrix-legend` - Легенда

## 🎮 Как использовать

### 1. Открыть секцию
```
Админка → Sidebar → "Управление ролями"
```

### 2. Изменить функциональное право
```
1. Найти функцию в таблице "ФУНКЦИОНАЛЬНЫЕ ПРАВА"
2. Кликнуть на статус роли (ON или OFF)
3. Право автоматически переключается и сохраняется
4. Уведомление об успехе
```

### 3. Изменить видимость раздела
```
1. Найти раздел в таблице "ВИДИМОСТЬ РАЗДЕЛОВ"
2. Кликнуть на статус роли
3. Раздел становится видимым/скрытым для этой роли
4. Автосохранение
```

### 4. Сбросить все права
```
1. Кликнуть "Сбросить к дефолтным"
2. Подтвердить действие
3. Все роли (кроме Super Admin) получат дефолтные права
```
