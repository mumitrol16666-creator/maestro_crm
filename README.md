# Maestro CRM

CRM для **музыкальной школы Maestro** (офлайн-школа: заявки, группы, абонементы, касса, расписание).

Онлайн-платформа курсов — отдельный проект: `maestro_school`.

## Что уже реализовано

- воронка заявок, причины потерь, возврат потерянных лидов и конверсия в ученика;
- ученики, дополнительные телефоны, семьи, группы и преподаватели;
- направления, тарифы, абонементы, продления, оплаты, долги и заморозки;
- регулярное и разовое расписание, кабинеты и проверка конфликтов;
- процесс офлайн-урока: преподаватель заполняет его в Learning Platform,
  администратор проверяет в очереди и подтверждает списание;
- касса, зарплаты, комиссии менеджеров и преподавателей;
- аналитика по продажам, ученикам, преподавателям, менеджерам и доходу;
- журнал действий и настраиваемая видимость разделов по ролям;
- SSO, создание/привязка аккаунтов учеников и преподавателей в Learning Platform;
- интеграционные API для расписания, уроков, посещаемости и абонементов.
- защищённая выдача списка учеников конкретного преподавателя для вкладки
  «Мои ученики» в Learning Platform.

## Роли

- `sales_manager` — заявки, ученики, продажи, оплаты и продления;
- `teacher` — ограниченный просмотр учеников и расписания;
- `admin` — ежедневная операционная и финансовая работа;
- `super_admin` — полный контроль, права, критические удаления и комиссии;
- `student` — собственный read-only кабинет CRM; основной учебный интерфейс
  находится в Learning Platform.

Подробный аудит: [IMPLEMENTATION_AUDIT_2026-06-19.md](./IMPLEMENTATION_AUDIT_2026-06-19.md).

## Запуск локально

```bash
cd backend
cp .env.example .env   # DATABASE_URL, JWT_SECRET
npm install
npx prisma db push
node scripts/init-maestro-config.js   # направления + кабинеты
node create-super-admin.js 77001234567 YourPassword
npm run dev

cd ../frontend
python3 -m http.server 8000
```

### Аварийная очистка тестовых операционных данных

Суперадминистратор может открыть раздел «Пользователи» и внизу воспользоваться блоком
«Аварийная очистка CRM». Перед удалением панель показывает количество затрагиваемых
записей и требует точную фразу `ОЧИСТИТЬ MAESTRO`.

Сервер сначала создаёт полный дамп PostgreSQL в `/var/backups/maestro-crm`. Если резервная
копия не создана, очистка отменяется. Удаляются абонементы, платежи, касса, зарплаты,
группы, занятия, посещаемость, заморозки и расписание; балансы учеников обнуляются.
Аккаунты, заявки, направления, тарифы, залы, роли и настройки сохраняются.

- CRM логин: `http://localhost:8000/public/login.html`
- API: `http://localhost:5001`
