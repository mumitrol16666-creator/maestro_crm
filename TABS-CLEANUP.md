## Как работает механика заморозок (без вкладки)

### 1. Ученик создает заморозку
```
Профиль ученика → Кнопка "Заморозить занятие"
                → Выбор занятий
                → POST /api/freezes
                → Заморозка создается (status: active или pending)
```

### 2. Заморозка автоматически работает
```
При отметке посещаемости:
  ├─ Проверяется isStudentFrozen()
  ├─ Если есть заморозка → чекбокс disabled
  ├─ Показывается ❄️ ЗАМОРОЗКА
  └─ Занятие не списывается
```

### 3. Отображение в профиле ученика
```
Секция "Активные заморозки":
  ├─ Показывает все активные заморозки
  ├─ Тип, период, количество занятий
  └─ Обновляется при создании новой
```

### 4. Отображение в карточке ученика (админка)
```
При просмотре профиля ученика:
  ├─ Показывается "Заморозок использовано: 1/2"
  ├─ История использования заморозок
  └─ Статус абонемента с учетом заморозок
```

## Почему удалены вкладки?

### 1. Не участвуют в основной логике
- Админу не нужно постоянно одобрять заморозки
- Обычные заморозки (`regular`, `period`) одобряются автоматически
- Заморозки видны в карточке ученика при необходимости

### 2. Упрощение интерфейса
- Меньше вкладок = проще навигация
- Фокус на основные функции
- Чище sidebar

### 3. Функциональность сохранена
- Ученики могут заморозить занятия
- Заморозки учитываются в посещаемости
- Статистика заморозок доступна в карточке
