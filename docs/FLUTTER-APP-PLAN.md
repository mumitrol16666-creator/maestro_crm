## 📋 ЧТО НУЖНО ДЛЯ ПУБЛИКАЦИИ

### iOS (App Store)

1. **Apple Developer Account** ($99/год)
   - Личная информация
   - Номер карты для оплаты
   - Подтверждение личности

2. **Apple ID**
   - Обычный Apple ID (бесплатно)

3. **Mac компьютер** (обязательно для сборки)
   - macOS 13+ (Ventura или новее)
   - Xcode 14+
   - Можно использовать облачные сервисы (Codemagic, GitHub Actions)

4. **Перед публикацией нужно:**
   - Иконка приложения (1024x1024 PNG)
   - Скриншоты для разных размеров экрана
   - Описание приложения (на нескольких языках)
   - Политика конфиденциальности (обязательно!)
   - Категория приложения
   - Возрастной рейтинг
   - Контакты поддержки

5. **Сертификаты и профили:**
   - Development Certificate
   - Distribution Certificate  
   - Provisioning Profiles
   - App ID

6. **Соглашения:**
   - Tax forms (W-8BEN для Казахстана)
   - Banking information (для получения платежей, если планируете платные функции)

### Android (Google Play)

1. **Google Play Developer Account** ($25 один раз)
   - Google аккаунт
   - Номер карты для оплаты
   - Подтверждение личности

2. **Перед публикацией нужно:**
   - Иконка приложения (512x512 PNG)
   - Скриншоты (минимум 2, рекомендуется 5-8)
   - Feature graphic (1024x500)
   - Описание приложения
   - Политика конфиденциальности (обязательно!)
   - Категория приложения
   - Возрастной рейтинг
   - Контакты поддержки

3. **Подписание приложения:**
   - Keystore файл для подписи APK/AAB
   - Очень важно сохранить! Без него нельзя обновлять приложение

4. **Сертификат подписи:**
   - Создается при первой сборке
   - Нужно надежно сохранить (восстановить невозможно)

## 🎯 СТРУКТУРА FLUTTER ПРИЛОЖЕНИЯ

### Что можно перенести из веб-версии:

1. **Экран авторизации** (`/api/auth/login`, `/api/auth/register`)
2. **Экран профиля:**
   - Личные данные (`/api/students/:id`)
   - Мои группы (`/api/groups`)
   - Абонемент (`/api/memberships`)
   - Ближайшие занятия (`/api/classes`)
   - Практики (`/api/practices`)
   - История посещений (`/api/bookings`)

3. **Функционал:**
   - Заморозка занятий (`/api/freezes`)
   - Продление абонемента
   - Принятие оферты
   - Редактирование профиля

## 🚀 ПРОЦЕСС ПУБЛИКАЦИИ

### iOS (App Store)

1. **Подготовка:**
   - Регистрация Apple Developer Account
   - Настройка сертификатов в Xcode
   - Подготовка материалов (скриншоты, описание)

2. **Сборка:**
   ```bash
   flutter build ios --release
   ```

3. **Загрузка через Xcode:**
   - Архив проекта
   - Валидация
   - Загрузка в App Store Connect

4. **App Store Connect:**
   - Заполнение информации о приложении
   - Загрузка скриншотов
   - Настройка цены и доступности
   - Отправка на модерацию

5. **Модерация:**
   - 1-3 дня (обычно)
   - Apple проверяет соответствие правилам

### Android (Google Play)

1. **Подготовка:**
   - Регистрация Google Play Developer Account
   - Создание keystore для подписи
   - Подготовка материалов

2. **Сборка:**
   ```bash
   flutter build appbundle --release
   ```

3. **Google Play Console:**
   - Создание приложения
   - Заполнение информации
   - Загрузка AAB файла
   - Загрузка скриншотов
   - Настройка цены и доступности
   - Отправка на проверку

4. **Проверка:**
   - 1-7 дней (обычно 1-3 дня)
   - Автоматическая проверка безопасности

## ⚠️ ВАЖНЫЕ МОМЕНТЫ

### Политика конфиденциальности
- **Обязательна** для обоих магазинов
- Должна описывать, какие данные собираете и как используете
- Должна быть доступна по ссылке

### Возрастные ограничения
- Ваше приложение: вероятно 12+ или 17+ (если есть оплаты)
- Нужно заполнить анкету в каждом магазине

### Локализация
- Google Play: можно публиковать только на русском/казахском
- App Store: можно выбрать доступные страны
- Рекомендуется хотя бы английская версия описания

### Обновления
- iOS: через App Store Connect
- Android: через Google Play Console
- Оба поддерживают автоматические обновления для пользователей

## 💡 РЕКОМЕНДАЦИИ

1. **Начните с Android:**
   - Дешевле ($25 vs $99)
   - Быстрее проходит проверку
   - Проще процесс публикации

2. **Используйте существующий API:**
   - Не нужно менять backend
   - Просто добавьте CORS заголовки если нужно
   - Мобильное приложение будет использовать те же endpoints

3. **Дизайн:**
   - Можно переиспользовать дизайн веб-версии
   - Адаптировать под мобильные экраны
   - Использовать Material Design или Cupertino

4. **Тестирование:**
   - Тестируйте на реальных устройствах
   - iOS симулятор хорош, но реальное устройство лучше
   - Android эмулятор тоже работает неплохо
