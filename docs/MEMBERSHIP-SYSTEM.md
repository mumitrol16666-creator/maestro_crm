# 💳 СИСТЕМА УПРАВЛЕНИЯ АБОНЕМЕНТАМИ

## 📋 ПРАВИЛА СТУДИИ

### **1. Абонемент:**
- **8 занятий в месяц**
- Занятие списывается автоматически если:
  - Было в расписании группы ученика
  - Ученик НЕ поставил заморозку
  - Независимо от того, пришёл или нет!

### **2. Заморозки (по занятиям, не по датам):**

#### **Бесплатные заморозки:**
- **Мужчины:** 1 заморозка за абонемент
- **Женщины:** 2 заморозки за абонемент
- Ученик САМ замораживает в "Мой профиль"
- Следующее занятие не списывается

#### **Специальные случаи (без лимита):**
- **Менструация (женщины):** автоматически одобряется
- **Командировка:** требует документ, одобрение админа
- **Болезнь:** требует больничный, одобрение админа

### **3. Дополнительные занятия:**
- Супер Админ и Админ могут добавить N занятий
- С указанием причины (компенсация, подарок и т.д.)

## 🗄️ СТРУКТУРА ДАННЫХ

### **Student (обновить):**
```javascript
{
  name: String,
  phone: String,
  email: String,
  gender: String, // 'male' или 'female' ← НОВОЕ!
  role: String,
  groups: [...],
  activeMembership: ObjectId,
  registeredAt: Date
}
```

### **Membership (обновить/создать):**
```javascript
{
  student: ObjectId,
  type: String, // 'trial', 'monthly', 'quarterly'
  
  // Занятия
  totalClasses: Number, // 8 для месячного
  classesRemaining: Number,
  classesUsed: Number,
  
  // Даты
  startDate: Date,
  endDate: Date,
  activatedAt: Date,
  
  // Заморозки
  freezesAvailable: Number, // 1 для М, 2 для Ж
  freezesUsed: Number,
  
  // История операций
  transactions: [{
    type: String, // 'deduct' (списание), 'add' (добавление), 'freeze' (заморозка)
    amount: Number,
    reason: String,
    date: Date,
    classId: ObjectId, // если связано с занятием
    freezeId: ObjectId, // если связано с заморозкой
    addedBy: ObjectId // если админ добавил вручную
  }],
  
  status: String, // 'active', 'expired', 'frozen'
  createdAt: Date,
  updatedAt: Date
}
```

### **Freeze (новая модель):**
```javascript
{
  student: ObjectId,
  membership: ObjectId,
  
  type: String, // 'regular' (бесплатная), 'period' (менструация), 
                // 'business_trip' (командировка), 'sick' (болезнь)
  
  // Заморозка по ЗАНЯТИЯМ (не датам!)
  frozenClasses: Number, // сколько занятий пропустит
  
  // Даты (для справки и расчетов)
  startDate: Date,
  endDate: Date,
  
  // Статус
  status: String, // 'pending' (ожидает одобрения), 'active', 'completed', 'rejected'
  
  // Документы (если нужны)
  documentUrl: String, // ссылка на файл
  documentNote: String,
  
  // Кто одобрил/отклонил
  processedBy: ObjectId,
  processedAt: Date,
  rejectionReason: String,
  
  createdAt: Date,
  updatedAt: Date
}
```

## ⚙️ ЛОГИКА РАБОТЫ

### **1. Создание абонемента:**

```
Админ конвертирует заявку в ученика:
↓
Создается Membership:
  - totalClasses: 8 (для месячного)
  - classesRemaining: 8
  - freezesAvailable: gender === 'male' ? 1 : 2
  - status: 'active'
  - startDate: сегодня (или выбранная дата)
  - endDate: startDate + 30 дней
```

### **2. Списание занятий (автоматическое):**

```
Препод отмечает посещаемость:
↓
Backend проверяет:
  1. Есть ли у ученика активный абонемент?
  2. Есть ли активная заморозка на это занятие?
  
Если НЕТ заморозки:
  ↓
  membership.classesRemaining -= 1
  membership.classesUsed += 1
  membership.transactions.push({
    type: 'deduct',
    amount: 1,
    reason: 'Занятие пройдено',
    classId: class._id,
    date: class.date
  })
  
Если ЕСТЬ заморозка:
  ↓
  Занятие НЕ списывается
  freeze.frozenClasses -= 1
  Если frozenClasses === 0:
    freeze.status = 'completed'
```

### **3. Заморозка ученика (в профиле):**

```
Ученик в "Мой профиль":
↓
Видит кнопку "ЗАМОРОЗИТЬ ЗАНЯТИЕ"
↓
Выбирает тип:
  • Бесплатная (если доступна)
  • Менструация (для женщин, автоматом)
  • Командировка (с документом)
  • Болезнь (с больничным)
↓
Указывает даты начала/окончания
↓
Система считает сколько занятий в этот период
↓
Создается Freeze со статусом:
  - 'active' для бесплатной и менструации
  - 'pending' для командировки/болезни
↓
Если 'pending' → админ одобряет в админке
```

### **4. Одобрение заморозки админом:**

```
Админ в разделе "ЗАМОРОЗКИ":
↓
Видит заявки со статусом 'pending'
↓
Кликает "ОДОБРИТЬ":
  freeze.status = 'active'
  freeze.processedBy = admin._id
  freeze.processedAt = Date.now()
  
ИЛИ "ОТКЛОНИТЬ":
  freeze.status = 'rejected'
  freeze.rejectionReason = "Причина..."
```

### **5. Добавление занятий админом:**

```
Админ в профиле ученика:
↓
Кнопка "ДОБАВИТЬ ЗАНЯТИЯ"
↓
Модалка:
  Количество: [5]
  Причина: [Компенсация за отмену занятия]
↓
membership.classesRemaining += 5
membership.totalClasses += 5
membership.transactions.push({
  type: 'add',
  amount: 5,
  reason: 'Компенсация за отмену занятия',
  addedBy: admin._id,
  date: Date.now()
})
```

## 💡 ПРИМЕРЫ СЦЕНАРИЕВ

### **Сценарий 1: Обычное использование**
```
1 октября:
  Иван покупает абонемент → 8 занятий

3 октября (занятие):
  Пришёл → -1 занятие → осталось 7

5 октября (занятие):
  Не пришёл → -1 занятие → осталось 6 (сгорело!)

7 октября:
  Заморозил следующее занятие (бесплатная)
  Осталось заморозок: 0/1

8 октября (занятие):
  Не пришёл, но есть заморозка → 0 → осталось 6 (не сгорело!)

31 октября:
  Осталось: 2 занятия
  Админка: оранжевый badge → звонят, предлагают продление
```

### **Сценарий 2: Женщина с менструацией**
```
1 октября:
  Мария покупает абонемент → 8 занятий, 2 заморозки

10 октября:
  Менструация → замораживает 2 занятия (10, 12 октября)
  Осталось заморозок: 0/2

10, 12 октября:
  Занятия не списываются → осталось 8

25 октября:
  Использовала только 6 занятий → осталось 2
  Все заморозки использованы
```

### **Сценарий 3: Командировка**
```
15 октября:
  Петр в командировке 20-25 октября
  Создает заявку на заморозку с документом
  Статус: 'pending'

16 октября:
  Админ видит badge [1] в "ЗАМОРОЗКИ"
  Открывает, смотрит документ
  Одобряет
  Статус: 'active'
  
20-25 октября:
  4 занятия не списываются
  После 25 октября заморозка 'completed'
```

### **Сценарий 4: Компенсация от студии**
```
18 октября:
  Занятие отменилось по вине студии
  
19 октября:
  Админ открывает профиль Ивана
  Кликает "ДОБАВИТЬ ЗАНЯТИЯ"
  Вводит: +1 занятие
  Причина: "Компенсация за отмену 18.10"
  
Результат:
  Было: 3 занятия
  Стало: 4 занятия
  История транзакций: "+1 (Компенсация...)"
```
