# Отчёт: аналитика и потери после пробного

Дата: 21 июня 2026 года.

## Исправленная логика

- Перевод заявки из пробного в `rejected` больше не стирает исторический факт пробного.
- Этап потери автоматически определяется как `after_trial`, если пробный урок уже завершён, использован пробный абонемент или есть подтверждённое посещение.
- Первоначальные дата и ответственный обработки заявки не перезаписываются поздним отклонением.
- В диалоге причины потери корректно применяется рекомендуемый этап.
- Конверсия учитывает только обычные абонементы, созданные после пробного.
- Потеря после пробного не считается раньше времени: в знаменатель входят завершившие 30-дневное окно решения либо явно отклонённые после пробного.
- Повторные пробные одного ученика в выбранном периоде не раздувают когорту.

## Улучшенная аналитика

- Добавлена воронка: пробные → посетили → купили абонемент → потеряны → ожидают решения.
- В карточке потерь отображается число клиентов, по которым окно решения ещё не завершилось.
- В разделе потерь появился список конкретных заявок с датой, этапом, причиной и ответственным.
- «Действующие ученики» теперь являются объединением активных пробных и постоянных учеников; «Постоянные» считаются отдельно.
- Подпись потерянных учеников приведена в соответствие с фактическим правилом: отсутствие оплат более трёх месяцев.
- Исправлена работа пользовательского диапазона дат без сдвига из-за часового пояса.

## Проверяемый сценарий

1. Создать заявку и оформить пробный.
2. Провести и подтвердить пробный урок.
3. Перевести заявку в `Отклонено`.
4. Указать причину потери.
5. Заявка должна попасть в аналитику с этапом `После пробного`, остаться в исторической когорте пробных и отобразиться в списке последних потерь.
