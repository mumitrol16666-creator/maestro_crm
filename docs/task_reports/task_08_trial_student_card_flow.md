# Отчёт: карточка ученика после пробного

## Что изменено

- Конвертация пробной заявки создаёт только карточку ученика.
- Абонемент, группа и платёж больше не создаются автоматически из заявки.
- В заявке отдельно сохраняются преподаватель пробного, кабинет, дата/время и факт оплаты диагностического урока 2000 ₸.
- При назначении автоматически создаётся один урок типа `trial` длительностью 60 минут.
- Этот урок сразу появляется в общем расписании и в запланированных уроках приложения выбранного преподавателя.
- Перенос пробного обновляет тот же урок, а не создаёт новый.
- После создания можно сразу открыть карточку и отдельно оформить абонемент и реальный платёж.
- Пробный урок не оплачивается преподавателю.
- Пол удалён из формы конвертации и формы абонемента.
- Пол добавлен в редактирование профиля ученика.
- Заморозки при создании абонемента рассчитываются по полу из профиля, но остаются настраиваемыми вручную.
- Аналитика пробных поддерживает обе схемы:
  - старые пробные с `trial`-абонементом;
  - новые пробные, где историей является конвертированная пробная заявка.
  - Потеря после пробного считается по окну 14 дней от назначенной даты пробного.
  - Закрытием считается только реальный платёж, поступивший на баланс ученика. Само создание карточки или абонемента продажей не считается.

## Ожидаемый сценарий

1. Провести пробный урок.
2. В заявке нажать создание ученика.
3. Получить карточку без автоматически созданного абонемента.
4. В карточке заполнить пол через «Редактировать».
5. В карточке отдельно создать нужный абонемент и внести платёж.
