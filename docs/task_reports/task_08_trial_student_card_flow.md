# Отчёт: карточка ученика после пробного

## Что изменено

- Конвертация пробной заявки создаёт только карточку ученика.
- Абонемент, группа и платёж больше не создаются автоматически из заявки.
- После создания можно сразу открыть карточку и отдельно оформить абонемент и возвратный депозит/платёж.
- Пробный урок не оплачивается преподавателю.
- Пол удалён из формы конвертации и формы абонемента.
- Пол добавлен в редактирование профиля ученика.
- Заморозки при создании абонемента рассчитываются по полу из профиля, но остаются настраиваемыми вручную.
- Аналитика пробных поддерживает обе схемы:
  - старые пробные с `trial`-абонементом;
  - новые пробные, где историей является конвертированная пробная заявка.
  - Потеря после пробного считается по окну 14 дней и учитывает оплату на балансе или абонемент.

## Ожидаемый сценарий

1. Провести пробный урок.
2. В заявке нажать создание ученика.
3. Получить карточку без автоматически созданного абонемента.
4. В карточке заполнить пол через «Редактировать».
5. В карточке отдельно создать нужный абонемент и отдельно отметить возвратный депозит/платёж.
